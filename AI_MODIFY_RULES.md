## 一、每次修改必须做的事（按顺序）

1. **修改代码**（功能/修复）
2. **更新 `CHANGELOG.md`** —— 在最上面加新版本的更新内容
3. **更新 `README.md`** —— 如果加了新配置项或新功能，必须写进去
4. **更新版本号**（在 `metadata.yaml`）—— 修改代码就要升版本号
   - 修复 bug → 升最后一位（v1.0.6 → v1.0.7）
   - 加小功能 → 升中间位（v1.0.7 → v1.1.0）
   - 大改动 → 升第一位（v1.x.x → v2.0.0）
5. **提交并推送到 GitHub** —— `git commit` + `git push origin main`
6. **打 stable 标签** —— `git tag vX.X.X-stable` + `git push origin vX.X.X-stable`

---

## 二、退回版本（用户说"退回去"）

- 用户说"退回去"三个字 = 退回到上一个 stable 标签
- 最近的稳定版本标签：`v1.0.11-stable`（2026-07-02，真正修复 Gemini 报 `value at top-level must be a list` 问题）
- 操作：
  1. `git checkout main`
  2. `git reset --hard v1.0.11-stable`
  3. `git push origin main --force`

---

## 三、沟通注意事项

- **用户是零基础新手**：解释技术问题时要简单易懂，尽量用比喻
- **不要假设用户懂 git / 编程术语**
- **重要改动用列表列清楚**，不要写大段文字
- **每次功能升级后都打 stable 标签**，方便随时退回

## 四、常见问题

### 插件配置里看不到新选项？
- 新配置项必须在 `_conf_schema.json` 里定义
- AstrBot 可能有缓存，试试"重载插件"或重启

### 版本号怎么升？
- 小修小补（改bug、优化）→ +0.0.1
- 加新功能 → +0.1.0
- 大改动不兼容 → +1.0.0
